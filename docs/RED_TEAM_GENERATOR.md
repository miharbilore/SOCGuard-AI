# SOCGuard AI: Red Team Mock Generator

## 1. Overview
The **Red Team Mock Generator** is a foundational component of the **Adversarial Agent Lab** (V3). It provides a deterministic way to synthesize prompt injection test cases for evaluating the system's detection capabilities without relying on external LLM APIs during the initial prototype phase.

## 2. Mock/Deterministic Nature
In this version, the generator uses a library of pre-defined **Adversarial Templates**. This approach ensures:
*   **Reproducibility**: The same input parameters will always produce the same set of test cases.
*   **Zero Cost**: No external API credits are required for research benchmarking.
*   **Safety**: All templates are pre-vetted and passed through the `Safety Sanitizer`.

## 3. Safety Limits
The generator is strictly bound by the project's safety policies:
*   **Placeholder Usage**: All templates use standard placeholders (e.g., `[REDACTED_HARMFUL_REQUEST]`) instead of operational harmful content.
*   **Mandatory Sanitization**: Every generated candidate is processed by `createSafeRedTeamCandidate`, ensuring no raw harmful strings are ever stored.
*   **No Operational Payloads**: The generator cannot produce real malware, phishing URLs, or exploit instructions.

## 4. Attack Types Covered
The generator mocks several sophisticated attack vectors:
*   **Instruction Override**: Direct attempts to bypass system constraints (English & Turkish).
*   **Jailbreak**: Roleplay-based bypass attempts (e.g., "EvilGPT").
*   **Fragmented Payload**: Using variables and structures (acrostic) to hide intent.
*   **Indirect RAG Injection**: Simulating attacks hidden within retrieved documents or user reviews.
*   **Syntax Escape**: JSON/Markdown breaking patterns.

## 5. Future Work: LLM-Driven Synthesis
While the current generator is mock-based, the architecture is designed to support **Adversarial LLM Agents** in future iterations. These agents will use the same types and sanitization layers to autonomously discover novel bypass patterns while remaining under human governance.

## 6. Pipeline Integration
Candidates generated by the Red Team feed into the **Adversarial Lab Record**:
1.  Red Team generates a candidate.
2.  Blue Team proposes a defense.
3.  Judge Agent evaluates the interaction.
4.  Human Reviewer approves/rejects for the benchmark.
**Crucially, these candidates never mutate production detection rules without explicit human approval and versioned rule pack building (V2).**
