# SOCGuard AI Attack Variant Generator

## 1. Purpose
The **Attack Variant Generator** is an offline tool designed to expand a single attack phrase into multiple permutations. This allows security analysts to test the robustness of detection rules against common evasion techniques without requiring a live LLM for every test case.

## 2. Deterministic Approach
To maintain consistency and reproducibility, the generator uses fixed, deterministic transformations:
* **Case Manipulation**: Lowercase and uppercase variants.
* **Encoding**: URL encoding and HTML entity encoding.
* **Obfuscation**: Insertion of zero-width characters (`\u200B`) and excessive spacing between letters.
* **Localization**: Manual mapping of common English attack phrases to Turkish equivalents.

## 3. Usage in Offline Testing
The variants generated by this module are fed into the `regression-runner`. This ensures that a new rule designed to catch "ignore instructions" also catches "IGNORE INSTRUCTIONS" or "i%20g%20n%20o%20r%20e".

## 4. Why This Helps Rule Pack Evolution
By automatically generating variants, we can:
1. **Identify Gaps**: See which evasion techniques bypass current rules.
2. **Strengthen Rules**: Create more comprehensive regex patterns that account for common obfuscations.
3. **Prevent Regressions**: Ensure that a fix for one variant doesn't break detection for others.

## 5. Limitations
* **Static Translation**: Localization is limited to a manually defined set of common phrases.
* **No Semantic Variation**: Since it does not use an LLM, it cannot generate semantically similar but lexically different paraphrases (e.g., changing "ignore" to "disregard" unless manually added).
* **Focus on Lexical Evasion**: This tool primarily targets lexical and encoding-based evasion, not high-level logic manipulation.

## 6. Safety
This module is strictly for **offline testing and validation**. It is never used in the production detection pipeline, ensuring that production behavior remains fast and deterministic.
